    System Requirements

    The software is a web application written in Perl CGI, and runs on a standard Unix/Linux system that's running webserver software such as Apache - the majority of commercial servers fit this discription - and also requires Perl 5.0 or better installed on the server (also usually always the case). The software can also be run on a Windows 95 or better system running a webserver. The Sambar Server can be used to modify the files on a windows computer, in conjunction with ActivePerl for Win32. In that case you have to remember to change the first few lines of the webtool.cgi file, before uploading it to your Unix/Linux server. For instance, while working on the software on a Windows machine, the first few lines of webtool.cgi look like this:

      #!c:\Perl\bin\perl.exe
      $db_use_flock = 0;
      $db_dir_url = "http://localhost/cgi-bin/newWebtool";

    The first line points to the directory where Perl is installed, while the second turns off file locking, which the Windows environment does not support. The last line points to the URL you use to view the software in a browser for offline development. For installing on a Unix system, however, the first few lines should be changed to:

      #!/usr/bin/perl
      $db_use_flock = 1;
      $db_dir_url = "http://yourDomain.com/cgi-bin/yourDirectory";

     
    Installing on Your Server

    To install the software, you need to find an ISP (or a friend who's already running a Unix/Linux webserver), and tell them you have some CGI scripts you want to install. If you already have web space rented with an ISP, it may not be enough - ISP's generally offer CGI access for a little more money than hosting regular, static webpages (and generally, ISP's offering free web space do not include the ability to run CGI).

    The costs for CGI access are still modest, however: usually around $10 / month or less (in U.S. dollars). For instance, the webtool demo was originally hosted on www.megs.com for $99 U.S. per year, which is about the usual fee. Once you've confirmed with your ISP that the account you've bought with them allows for running CGI scripts, you can take the following steps to install the software:

    1) Download the software's source files. These files come packaged as a single, zipped-up file. You'll need a utility like Winzip to extract all the files, which are shown below. Keep all the files together in a single directory (say, c:\TEMP_webtool).

    2) You'll need a text editor that can save files in Unix format. I use Programmer's File Editor, which is a good free utility. Many HTML editors like Homesite also let you save files in Unix format. Important: the files shown contain the source code for the software. However, remember not to open them using Notepad (or any other text editor that can only save files in Windows format), or you may inadvertently save any changes you make in the wrong file format. When you upload them to a Unix/Linux system, files altered using Notepad will not run.

    3) You'll need to download and install an FTP program such as WS_FTP or Cute FTP. These programs typically come with a 30-day trial period, and are used to upload files to your server. They also perform the crucial step of changing file permissions using the CHMOD command in Unix. You'll need to CHMOD each file you've uploaded (usually you right-click on the file), setting its read/write/execute properties correctly (a table showing the correct values is given later).

    4) Using your text editor, open up the webtool.cgi file, and change the following line (which is found near the top):

       $db_dir_url = "http://yourDomain.com/cgi-bin/webtool";

    This line needs to be changed to reflect the domain you're running the software in. Save the file in Unix format.

    5) To install, first connect to the web, then use your FTP program to login to your webserver (your ISP will have given you the username and password you need to login). Find the cgi-bin directory on your server, and create a subdirectory called webtool under it, and two directories under that, called userSession and images. Upload all the webtool files you previously extracted with your zip utility, into the webtool directory, and all files found in images into the corresponding server directory (the userSession directory should be empty). Finally, right-click on each file, to change the file permissions on each one, according to the following table:

    webtool.cgi (755) -rwxr-xr-x
    files ending in .pl or .cfg (644) -rw-r--r--
    files ending in .def .log .db .count (666) -rw-rw-rw-
    the userSession directory (777) drwxrwxrwx



    Test out your installation by pointing your browser at:

    http://yourDomain/cgi-bin/webtool/webtool.cgi

    You should see the login screen.
